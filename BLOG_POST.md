# tmuxとClaude Codeで構築する「完全自律型AI企業」の実験記録

![AI Corp Concept](https://img.shields.io/badge/AI-Corp-Genesis-blue)

2015年のMacBook ProをLinuxサーバー化し、そこで「AI社員」たちが働く仮想企業 **AI-Corp Genesis** を構築してみました。
tmuxとClaude Codeを組み合わせ、CEO、PM、エンジニア、法務、広報といった役割を持つエージェントが、ファイルシステムを通じて会話しながらプロジェクトを進める実験の記録です。

## 🏗 アーキテクチャ概要

### 1. 組織図 (The Org Chart)
tmuxのウィンドウを「部署」に見立て、以下のような構成で常駐させています。

*   **Window 0: 経営戦略室 (HQ)**
    *   **CEO:** 全体の指揮、予算管理
    *   **COO:** 整合性チェック
*   **Window 1: 技術開発部 (Dev)**
    *   **PM:** 仕様策定
    *   **Engineers:** 実装担当
*   **Window 2: 管理部 (Admin)**
    *   **Legal:** ライセンス・規約チェック
    *   **Knowledge:** 社内Wiki整備
*   **Window 3: マーケティング (Marketing)**
    *   **PR:** ブログ・SNS記事作成

### 2. 通信プロトコル
エージェント同士は直接チャットするのではなく、**ファイルシステム**を介して連携します。

> **Flow:** CEOが指示書を書く (`inbox/`) → PMが拾って仕様書を書く (`data/specs/`) → エンジニアがコードを書く (`src/`) → 法務がチェック → 広報がリリース記事を書く

これにより、各エージェントが非同期に動き、まるで本当のオフィスのように機能します。

## 🛠 技術スタック
*   **OS:** Linux (Ubuntu Server on MacBook Pro 2015)
*   **Middleware:** tmux (画面分割・セッション維持)
*   **AI Core:** Claude Code (Anthropic)
*   **Environment:** Docker / Python
*   **Networking:** Tailscale (外部からのセキュアアクセス)

## 💭 やってみて思ったこと (Dev Log)

### 箱庭としての面白さ
最初はあまり流れがわからず、Geminiに言われるがままに1つずつ部署を起動していましたが、4つくらいできたあたりで感覚が変わりました。

CEOが出した指示に対して、部下が完了報告を上げ、次のタスクを待っている様子……。
これが「箱庭」っぽくて非常に面白いのです。というか、健気でかわいい。

自分のカウンターパートは基本的に「CEOのウィンドウ」だけです。そこで「こういうアプリ作りたい」と投げると、あとはCEO役のAIがメッセージを作成し、各エージェントのInboxに放り込んでいきます。

### リソースとコストの現実
今回は合計6個くらいの部署（エージェント）を立ち上げました。
Claude Proプランだと、確かにすぐにレートリミットの上限になり、待ち時間が多く発生します。まともに自律作業させるには、Maxプランへの切り替えやAPI利用が必要になるレベルです。

「どの程度ちゃんと動かせるのか？」を知りたくて作った環境ですが、常時稼働で置いておけるVPS上にClaude Codeを入れて使うという選択は正解でした。手元のPCだと電源を切ってしまいますが、サーバー上のtmuxセッションなら、待ち時間が発生してもそのまま放置し、リセットされたら再開すればいいだけです。

### 「バイブコーディング」の実感
サーバー構築や設定も、Claude CodeやGemini CLI、Codex CLIで行いました。Tailscale経由でiPhoneからもアクセスできる環境が整っているので、組み合わせれば「スマホだけで開発指示を出し、帰宅したら動いている」という体験が可能です。

最近よく聞く「バイブコーディング（Vibe Coding）」という言葉。
最初は半信半疑でしたが、こうして環境が整うと、確かにスマホ片手にAIに指示出しするだけでモノができる感覚は、だいぶ実感が持てるようになりました。

### 自前サーバーの意義
単純にブログや画像生成、音楽生成のワークフローを組むだけなら、Difyやn8nなどのSaaSだけでも完結します。
しかし、権利関係を気にせずコンテンツ制作のワークフローを組もうとしたり、大量のデータを扱おうとすると、自前で機器もデータも準備する必要が出てきます。

全て自宅に置いて管理するのはスマートではない気もしましたが、今回のように「AI組織」を丸ごと実験場として飼っておくには、自前のLinux環境は最強のサンドボックスだと感じました。
